# 🌐 DO2: Linux Network

## 📌 О проекте
Проектирование, развертывание и администрирование сегментированной компьютерной сети на базе ОС Linux. Проект охватывает полный цикл сетевой настройки: от расчета IP-адресации и статической маршрутизации до настройки межсетевых экранов (iptables), служб DHCP, механизмов NAT (SNAT/DNAT) и безопасных SSH-туннелей.

## 🛠 Стек технологий и инструменты
- **ОС:** Ubuntu 20.04 Server LTS (5 виртуальных машин)
- **Сетевая конфигурация:** `netplan`, `ip`, `ipcalc`, `sysctl`
- **Маршрутизация и диагностика:** `ping`, `traceroute`, `tcpdump`, `netstat`, `nmap`, `iperf3`, `telnet`
- **Безопасность и фильтрация:** `iptables` (filter, nat), SSH Tunnels (Local/Remote forwarding)
- **Сетевые службы:** `isc-dhcp-server`, `apache2`

## ⚙️ Детальное описание выполненных задач

- **Part 1. Инструмент ipcalc:**

  Проанализировал и рассчитал сетевые параметры с помощью утилиты `ipcalc`. Определил адреса сетей, выполнил конвертацию масок подсети между десятичным, префиксным и двоичным форматами. Рассчитал диапазоны доступных хостов, классифицировал IP-адреса на публичные и частные (RFC 1918), а также определил корректность адресов шлюзов для заданных подсетей.

- **Part 2. Статическая маршрутизация между двумя машинами:**

  Развернул две виртуальные машины и настроил статическую IP-адресацию через `netplan`. Организовал двустороннюю связность между разнородными подсетями: сначала путем ручного добавления маршрутов через `ip route add`, а затем сделал конфигурацию постоянной, внедрив правила маршрутизации непосредственно в YAML-файлы `netplan`. Проверил связность утилитой `ping`.

- **Part 3. Утилита iperf3:**

  Провел тестирование пропускной способности канала связи между двумя хостами с помощью `iperf3`. Выполнил расчеты и конвертацию единиц измерения скорости передачи данных (Mbps, MB/s, Kbps, Gbps) для корректной интерпретации результатов нагрузочного тестирования сети.

- **Part 4. Сетевой экран (iptables, nmap):**

  Разработал и применил скрипты межсетевого экранирования на базе `iptables`. Реализовал и сравнил две стратегии фильтрации: "запретить всё, кроме разрешенного" (default-deny) и "разрешить всё, кроме запрещенного" (default-allow). Настроил правила для открытия портов SSH (22) и HTTP (80), а также динамически блокировал и разрешал ICMP Echo Reply. Подтвердил доступность хоста и открытых портов через `nmap`, даже когда стандартный `ping` был заблокирован.

- **Part 5. Статическая маршрутизация сети:**

  Спроектировал и развернул многосегментную сеть из 3 рабочих станций и 2 роутеров. Включил IP-форвардинг на уровне ядра (`sysctl -w net.ipv4.ip_forward=1`) и закрепил его в `sysctl.conf`. Настроил шлюзы по умолчанию на рабочих станциях и прописал сложные статические маршруты на роутерах для обеспечения сквозной связности. Проанализировал прохождение пакетов с помощью `traceroute` и `tcpdump`, а также задокументировал генерацию ICMP-сообщений при попытке доступа к несуществующим узлам.

- **Part 6. Динамическая настройка IP с помощью DHCP:**

  Развернул и настроил службу `isc-dhcp-server` на роутерах для автоматической выдачи IP-адресов, шлюзов и DNS-клиентам. Реализовал статическое резервирование адресов (DHCP reservation) на основе MAC-адресов конкретных хостов. Протестировал процесс получения, обновления (renew) и освобождения (release) IP-адресов клиентами.

- **Part 7. NAT (SNAT / DNAT):**

  Реализовал механизмы преобразования сетевых адресов с помощью таблицы `nat` в `iptables`. Настроил SNAT (Masquerade) для обеспечения доступа внутренних хостов во внешнюю сеть. Настроил DNAT (Port Forwarding) для проброса внешнего порта 8080 роутера на внутренний порт 80 веб-сервера Apache. Успешно протестировал оба сценария, установив TCP-соединения через `telnet`.

- **Part 8. Дополнительно. Знакомство с SSH Tunnels:**

  Организовал безопасный доступ к сервисам, закрытым от прямого сетевого доступа (Apache, слушающий только `localhost`). Настроил и протестировал механизмы Local TCP Forwarding (проброс порта с локальной машины на удаленную) и Remote TCP Forwarding (обратный проброс порта с удаленной машины на локальную), проверив работоспособность туннелей через `telnet`.
