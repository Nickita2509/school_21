# 📈 DO4: Linux Monitoring v2.0 & Real-time Analytics

## 📌 О проекте
Комплексное решение для генерации нагрузки, парсинга логов и мониторинга состояния системы в реальном времени. Проект охватывает создание bash-скриптов для управления файловой системой, анализ веб-логов через `awk` и `GoAccess`, а также развертывание полноценного стека мониторинга на базе `Prometheus` и `Grafana` с созданием кастомных и готовых дашбордов.

## 📂 Структура проекта и артефакты
Проект разделен на две логические зоны в зависимости от типа артефактов:

**🛠 Исполняемые скрипты (находятся в папке `src/`):**
- 📁 **`src/01/`** — Генератор файлов с валидацией параметров и контролем свободного места.
- 📁 **`src/02/`** — Скрипт стресс-тестирования файловой системы (генерация вложенных структур до 100 уровней).
- 📁 **`src/03/`** — Утилита очистки системы (по лог-файлу, по таймстампу или по маске имени).
- 📁 **`src/04/`** — Генератор синтетических логов Nginx (формат combined).
- 📁 **`src/05/`** — Парсер логов на базе `awk` для извлечения метрик и ошибок.

**📊 Отчеты и конфигурации (находятся в корневых папках заданий):**
- 📁 **`src/06/`** → 📄 `report_task_6.md` (Настройка и скриншоты GoAccess).
- 📁 **`src/07/`** → 📄 `report_task_7.md` (Установка Prometheus/Grafana, кастомный дашборд, тесты `stress`).
- 📁 **`src/08/`** → 📄 `report_task_8.md` (Импорт готового дашборда Node Exporter, тесты `iperf3`).

## 🛠 Стек технологий и инструменты
- **Автоматизация:** Bash, `awk`, `grep`, `find`, `cron`
- **Анализ логов:** GoAccess, Nginx (combined log format)
- **Мониторинг:** Prometheus, Grafana, Node Exporter
- **Нагрузочное тестирование:** `stress`, `iperf3`, кастомные bash-скрипты
- **ОС:** Ubuntu 20.04 Server LTS

## ⚙️ Детальное описание выполненных задач

- **Part 1. Генератор файлов:**

  Разработал скрипт для создания структурированных тестовых данных на основе 6 входных параметров. Реализовал строгую валидацию ввода, генерацию имен папок и файлов с сохранением порядка символов и добавлением даты (DDMMYY). Внедрил механизм безопасности: скрипт автоматически прерывает работу, если свободное место в корневом разделе (`/`) опускается ниже 1 ГБ. Все действия логируются с указанием пути, даты и размера.

- **Part 2. Засорение файловой системы:**

  Написал скрипт для стресс-тестирования дисковой подсистемы. Реализовал генерацию случайного количества файлов (до 100 уровней вложенности) в произвольных директориях (с исключением системных `bin`/`sbin`). Добавил жесткий лимит остановки при достижении 1 ГБ свободного места (контроль через `df -h /`). В лог и в стандартный вывод добавлено точное время начала, окончания и общее время выполнения скрипта.

- **Part 3. Очистка файловой системы:**

  Создал утилиту для безопасного отката изменений, созданных в Part 2. Реализовал три независимых сценария очистки, выбираемых через параметр: 1) удаление на основе ранее сгенерированного лог-файла; 2) удаление по временному интервалу создания (с точностью до минуты); 3) удаление по маске имени файла (символы + нижнее подчеркивание + дата).

- **Part 4. Генератор логов:**

  Разработал скрипт для создания 5 реалистичных файлов логов Nginx (формат combined), каждый из которых имитирует трафик за один день (100–1000 записей). Реализовал рандомизацию корректных IP-адресов, HTTP-методов (GET, POST, PUT, PATCH, DELETE), кодов ответа (2xx, 4xx, 5xx) и User-Agent. В коде добавлены подробные комментарии с расшифровкой каждого используемого HTTP-статуса.

- **Part 5. Мониторинг (Парсинг логов):**

  Написал аналитический bash-скрипт на базе утилиты `awk` для обработки сгенерированных логов. В зависимости от переданного параметра (1–4) скрипт эффективно извлекает и сортирует данные: все записи по коду ответа, список уникальных IP-адресов, выборку всех запросов с ошибками (4xx/5xx) и список уникальных IP, с которых приходили ошибочные запросы.

- **Part 6. GoAccess:**

  Настроил утилиту GoAccess для визуализации тех же метрик, что и в Part 5, но в реальном времени. Сконфигурировал вывод как в терминальном интерфейсе, так и в виде интерактивной HTML-страницы, доступной через веб-браузер на локальной машине.

- **Part 7. Prometheus и Grafana:**

  Развернул и настроил стек мониторинга Prometheus + Grafana. Создал кастомный дашборд для визуализации ключевых метрик в реальном времени: загрузка CPU, доступная оперативная память, свободное дисковое пространство и операции ввода-вывода (I/O). Провел корреляционный анализ: запустил скрипт из Part 2 и утилиту `stress`, зафиксировав соответствующие пики на графиках дашборда.

- **Part 8. Готовый дашборд и сетевая нагрузка:**

  Импортировал и настроил официальный дашборд *Node Exporter Quickstart* из библиотеки Grafana Labs. Для тестирования сетевых метрик развернул вторую виртуальную машину в той же сети. Провел нагрузочное тестирование канала с помощью утилиты `iperf3`, проанализировав отображение сетевой активности (Network Traffic) на дашборде.

## 🚀 Как проверить

1. **Для проверки скриптов (Parts 1-5):** Перейдите в директорию `src/0x`, выдайте права `chmod +x main.sh` и запустите скрипт с параметрами, описанными в комментариях внутри кода.
2. **Для проверки отчетов (Parts 6-8):** Откройте файлы `report_task_X.md` в соответствующих папках.
