# 📊 DO3: Linux Monitoring & Bash Scripting

## 📌 О проекте
Разработка модульных Bash-скриптов для автоматизации сбора системных метрик, кастомизации терминального вывода и глубокого аудита файловой системы. Проект демонстрирует навыки написания структурированного, защищенного от некорректного ввода кода, парсинга конфигурационных файлов и работы с текстовыми потоками в Linux.

## 📂 Структура проекта
Все скрипты декомпозированы и размещены в директориях, соответствующих номерам заданий. Основной файл для запуска в каждой папке — `main.sh`.

- 📁 **`01/`** — Валидация входных параметров.
- 📁 **`02/`** — Сбор системных метрик и интерактивное сохранение отчета.
- 📁 **`03/`** — Визуальное оформление вывода через аргументы командной строки.
- 📁 **`04/`** — Чтение настроек из конфигурационного файла с fallback-логикой.
- 📁 **`05/`** — Глубокий рекурсивный аудит файловой системы.

---

## 🛠 Стек технологий и инструменты
- **Язык:** Bash (Shell scripting)
- **ОС:** Ubuntu 20.04 Server LTS
- **Обработка данных:** `awk`, `grep`, `sed`, `find`, `du`, `md5sum`, `sort`
- **Системные утилиты:** `hostname`, `timedatectl`, `uptime`, `ip`, `free`, `df`, `date`, `time`
- **Визуализация:** ANSI escape codes (управление цветом в терминале)

## ⚙️ Детальное описание выполненных задач

- **Part 1. Проба пера (Валидация ввода):**

  Разработал базовый Bash-скрипт, принимающий один текстовый параметр. Реализовал строгую валидацию входных данных: скрипт анализирует тип переданного аргумента и блокирует выполнение с выводом сообщения об ошибке, если передано числовое значение, обеспечивая корректную работу исключительно с текстом.

- **Part 2. Исследование системы:**

  Написал скрипт для агрегации ключевых метрик ОС в единый форматированный отчет. Извлек данные о хосте, временной зоне, пользователе, версии ОС, аптайме (в секундах и человеко-читаемом формате), сетевых параметрах (IP, маска, шлюз) и использовании ресурсов. Реализовал точное форматирование чисел: 3 знака после запятой для RAM (GB) и 2 знака для дискового пространства (MB). Добавил интерактивный диалог (Y/N) для сохранения отчета в файл с динамическим именем формата `DD_MM_YY_HH_MM_SS.status`.

- **Part 3. Визуальное оформление вывода:**

  Модернизировал скрипт из Part 2, добавив поддержку цветного вывода через ANSI-escape последовательности. Скрипт принимает 4 числовых аргумента (от 1 до 6) для независимой настройки фона и цвета шрифта для названий параметров (столбец 1) и их значений (столбец 2). Внедрил логическую проверку на коллизию: скрипт перехватывает попытку задать одинаковый цвет для фона и текста в пределах одного столбца, выводит описание проблемы и корректно завершает работу.

- **Part 4. Конфигурирование визуального оформления:**

  Улучшил архитектуру и UX, перенеся настройки цветов из аргументов командной строки во внешний конфигурационный файл. Реализовал парсинг файла с fallback-логикой: если ключ отсутствует или файл не найден, скрипт автоматически подставляет дефолтную цветовую схему. После вывода метрик скрипт печатает итоговую примененную схему для полной прозрачности настроек.

- **Part 5. Исследование файловой системы:**

  Разработал аналитический скрипт для глубокого аудита указанной директории (путь передается как параметр, обязательно заканчивающийся на `/`). Скрипт рекурсивно анализирует файловую систему и вычисляет: общее количество папок и файлов, топ-5 самых тяжелых директорий, детальную статистику по типам файлов (.conf, .txt, исполняемые, .log, архивы, симлинки), топ-10 самых тяжелых файлов и топ-10 самых тяжелых исполняемых файлов с вычислением их MD5-хешей. Добавил точный замер и вывод общего времени выполнения скрипта в секундах.

## 🚀 Как проверить и запустить

Для проверки перейдите в соответствующую папку `0x` и запустите `main.sh`. Убедитесь, что файлу выданы права на выполнение (`chmod +x main.sh`).

1. **Part 1:** `./main.sh "твой_текст"` (попробуйте передать число для проверки валидации).
2. **Part 2:** `./main.sh` (ответьте `Y` для проверки создания файла `.status`).
3. **Part 3:** `./main.sh 1 3 4 5` (попробуйте ввести `./main.sh 1 1 2 2` для проверки защиты от коллизии цветов).
4. **Part 4:** Создайте файл `config.txt` с параметрами (или запустите без него для проверки дефолтных значений) и выполните `./main.sh`.
5. **Part 5:** `./main.sh /var/log/` (рекомендуется запускать на директории с умеренным количеством файлов для демонстрации скорости работы).
