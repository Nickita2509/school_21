# 🐧 DO1: Linux (Ubuntu Server)

## 📌 О проекте
Развертывание и комплексное администрирование операционной системы Ubuntu 20.04 Server LTS в среде виртуализации. Проект охватывает полный цикл базовой настройки сервера: от установки и управления пользователями до настройки сети, обеспечения безопасности, мониторинга ресурсов и автоматизации задач.

## 🛠 Стек технологий и инструменты
- **ОС:** Ubuntu 20.04 Server LTS (CLI, без графического интерфейса)
- **Сети:** `netplan`, `ping`, `netstat`, DHCP, статическая маршрутизация
- **Мониторинг:** `top`, `htop`, `fdisk`, `df`, `du`, `ncdu`
- **Безопасность и доступ:** `sudo`, `sshd` (кастомный порт), `auth.log`
- **Автоматизация:** `cron`, `systemd`
- **Текстовые редакторы:** VIM, Nano, Mcedit

## ⚙️ Детальное описание выполненных задач

- **Part 1. Установка ОС:**

  Развернул виртуальную машину с Ubuntu 20.04 Server LTS в среде VirtualBox. Произвел чистую установку системы без графического интерфейса (GUI) для минимизации потребления ресурсов. Версию и кодовое имя релиза подтвердил через чтение системного файла `cat /etc/issue`.
- **Part 2. Создание пользователя:** 

  Создал дополнительного пользователя для повседневной работы. Добавил его в группу `adm` для предоставления прав на чтение системных журналов. Наличие пользователя и его параметры подтвердил через анализ файла `/etc/passwd`.
- **Part 3. Настройка сети ОС:** 

  Настроил базовые сетевые параметры: задал hostname (`user-1`) и актуальную временную зону. Проанализировал сетевые интерфейсы, уделив внимание назначению loopback-интерфейса `lo` (127.0.0.1). Получил адресацию через DHCP (Dynamic Host Configuration Protocol), определил внешний IP и шлюз по умолчанию. Перевел сетевой интерфейс на статическую конфигурацию, прописав статический IP, шлюз и публичные DNS-серверы (1.1.1.1, 8.8.8.8) через `netplan`. После перезагрузки проверил сетевую связность: успешный пинг до 1.1.1.1 и ya.ru с результатом `0% packet loss`.
- **Part 4. Обновление ОС:** 

  Провел аудит и обновление системных пакетов до актуальных версий с помощью `apt update && apt upgrade`. Убедился в отсутствии уязвимостей и pending-обновлений, выполнив команду повторно.
- **Part 5. Использование sudo:** 

  Настроил безопасное делегирование привилегий: добавил пользователя в группу `sudo` для выполнения административных задач без использования root-аккаунта. Продемонстрировал работу механизма, изменив hostname системы от имени обычного пользователя с префиксом `sudo`.
- **Part 6. Служба времени:** 

  Обеспечил корректность системного времени, настроив службу NTP (Network Time Protocol). Проверил текущее время часового пояса и подтвердил успешную синхронизацию со стратум-серверами через вывод `timedatectl show` (параметр `NTPSynchronized=yes`).
- **Part 7. Текстовые редакторы:** 

  Установил и настроил три текстовых редактора: VIM, Nano и Mcedit. Провел сравнительное тестирование: создал файлы `test_X.txt`, отработал сценарии сохранения (`:wq`, `Ctrl+O/X`) и отмены изменений (`:q!`, `Ctrl+C`). Освоил навигацию, поиск по содержимому и массовую замену строк в каждом из редакторов.
- **Part 8. Настройка SSHD:** 

  Развернул OpenSSH-сервер (`sshd`) для безопасного удаленного доступа. Включил автозапуск службы через `systemctl enable`. В целях безопасности (security hardening) изменил стандартный порт SSH с 22 на **2022**. Проанализировал фоновые процессы через `ps` с подобранными ключами. После перезагрузки подтвердил прослушивание кастомного порта на всех интерфейсах (`0.0.0.0:2022`) через `netstat -tan`, расшифровав значения ключей и столбцов вывода.
- **Part 9. Утилиты top и htop:** 

  Провел мониторинг производительности системы. С помощью `top` снял метрики: uptime, load average, загрузку CPU/RAM, определил PID процессов, потребляющих максимум ресурсов. В `htop` настроил кастомное отображение: отсортировал список по PID/CPU/MEM/TIME, отфильтровал процесс `sshd`, нашел `syslog` через встроенный поиск, добавил в шапку вывод hostname, clock и uptime.
- **Part 10. Утилита fdisk:** 

  Провел аудит блочных устройств с помощью `fdisk -l`. Снял параметры жесткого диска: название устройства, общий объем, количество секторов и размер выделенного swap-раздела.
- **Part 11. Утилита df:** 

  Оценил использование дискового пространства. С помощью `df` снял метрики корневого раздела `/` (размер, занято, свободно, процент использования) и определил единицы измерения. Через `df -Th` получил те же данные в человекочитаемом формате и определил тип файловой системы (ext4).
- **Part 12. Утилита du:** 

  Проанализировал потребление места директориями. С помощью `du` вывел размер папок `/home`, `/var`, `/var/log` в байтах и человекочитаемом виде (`-h`). Детализировал вывод для `/var/log`, используя шаблон `*` для оценки каждого вложенного элемента.
- **Part 13. Утилита ncdu:** 

  Установил интерактивную утилиту `ncdu` для более наглядного анализа дискового пространства. Просканировал директории `/home`, `/var`, `/var/log`, перекрестно сверив полученные данные с результатами работы `du`.
- **Part 14. Системные журналы:** 

  Провел диагностику системы через анализ системных журналов. Изучил `/var/log/dmesg` (загрузка ядра), `/var/log/syslog` (общие события) и `/var/log/auth.log` (аутентификация). Выявил время, имя пользователя и метод (SSH/TTY) последней успешной авторизации. Перезапустил службу SSHd и нашел соответствующую запись о рестарте в логах.
- **Part 15. Планировщик CRON:** 

  Настроил автоматизацию рутинных задач через планировщик `cron`. Добавил задание на выполнение команды `uptime` каждые 2 минуты. Подтвердил корректность работы, найдя строчки выполнения в системных логах. Вывел список активных задач (`crontab -l`) и успешно очистил расписание.
